What is Cyon

Cyon is a hacker-themed desktop environment, Python tool suite, Minecraft server platform, and scripting engine — all built from scratch in C, Python, Java, and JavaScript. It runs a full X11 window manager, a VTE terminal with a live system visualizer, a pixel art editor, a video editor, a dual-pane file manager, a 3D launcher world, local AI, a Discord bot, a Flask web dashboard, a Mineflayer combat bot, and a complete suite of custom Paper plugins — including CyonScript, a programming language built into Minecraft books.

// DESIGN PHILOSOPHYNo bloat. No Electron. No cloud. No subscriptions. Compiled C binaries for the desktop layer, Python for scriptable tools, Java for the server, JavaScript for the bot. Everything runs on your machine. No data leaves. No snitching occurs.

CruxibleBooks — CyonScript

The flagship plugin. CyonScript is a programming language written inside Minecraft books and executed as server commands. Scripts are saved as .txt files, loaded back as written books, and can run manually, on a schedule, or triggered by physical blocks anywhere in the world.

Commands

CommandDescription
/exportbook [name]Save held written book to file (default: .txt, any extension supported)
/importbook <name>Load a saved script as a written book
/listbooksList all saved script files
/runbook [player]Execute held book. Ops can run as any player.
/sendbook <player>Send book as bottle message — delivers on login if offline
/mailboxCheck and collect pending bottle messages
/linkblock <book> [op]Link looked-at block (button/plate/lever) to a script
/unlinkblockUnlink looked-at block
/listlinksShow all linked blocks
/cyfihere <n> [cooldown <s>]Register looked-at block as a named CyFi signal point
/cyfipoint add/remove/listManage CyFi points by coordinates

Language — Instructions

say <text> # broadcast to all players msg <text> # send message to yourself cmd <command> # run as player opcmd <command> # run as console (op-signed book + op runner) tp <x> <y> <z> # teleport give <item> [amount] # give item to self drop <player> <item> # drop item at player feet wait <ticks> # pause — 20 ticks = 1 second repeat <n> <instruction># repeat up to 100 times cyfi <point> on|off|pulse [ticks] # fire wireless redstone signal

Language — Variables + Math

# Built-in variables {player} {world} {x} {y} {z} {health} {food} # Custom variables and math set total 50 set half {total} / 2 # operators: + - * / % give diamond {half} # math works anywhere a number is expected tp {x} + 10 {y} {z} # including inside coordinates wait {total} * 20

Language — Control Flow

# if / else / endif — operators: == != > < >= <= if {health} < 10 say Low HP! else say HP ok endif # while / endwhile — capped at 100 iterations set x 5 while {x} > 0 say countdown: {x} set x {x} - 1 endwhile # for / endfor — optional step for i 1 to 10 step 2 say {i} endfor # this is a comment

CyFi — Wireless Redstone

Register named trigger points anywhere on the map. CyonScript fires them remotely — no physical wire needed. Players have cooldowns; ops bypass them.

# Op: look at empty block next to your mechanism /cyfihere lights cooldown 5 # In your script: cyfi lights pulse 20 # Link to a button anywhere on the map: /linkblock lights_script # Anyone clicking that button fires the script, which triggers the mechanism.

Scheduled Scripts

# config.yml — runs scripts at set times every day scheduled-books: - book: daily_news time: "08:00" - book: night_warn time: "20:00"

⚠ .bashrc Guard

cyon_neo sets CYON_NEO=1 before sourcing .bashrc. If your .bashrc launches an interactive program — a menu, cyon_cli, a custom loop — wrap it:

// WITHOUT THIS GUARDOpening a terminal inside cyon_neo will recurse. The shell loops until it freezes. Add the guard once and it never happens again.
if [ -z "$CYON_NEO" ]; then # your interactive menu / cyon_cli launch / program here fi # Everything outside the if block runs normally in both contexts

Uninstall

Select option 17 from compile_cyon. Removes: desktop launchers, ~/pyra_env, GTK theme (restores backup), Pyra PATH from ~/.bashrc, and the entire ~/cyon/ directory.

// CANNOT BE UNDONEComplete removal. The uninstaller asks once. It means it.
# apt packages are NOT removed automatically sudo apt remove build-essential libgtk-3-dev libvte-2.91-dev pkg-config yt-dlp ffmpeg
