▸ CYON

Hybrid GTK + CLI desktop tool — C / Python / Ollama / Discord

⚠ ACTIVE DEVELOPMENT

What is Cyon

Cyon is a lightweight, hacker-themed desktop control panel built in C with GTK 3. It runs a persistent GTK window that manages background processes, provides a command input, and launches a suite of Python utility tools. It is designed to be fast, modular, and extendable — living entirely inside ~/cyon/.

At its core, Cyon is a process manager: it spawns and controls three key services from a single UI — the Ollama LLM server, the local AI chatbot, and a Discord bot — all wired to a shared log view and input field.

// DESIGN PHILOSOPHY No bloat. No Electron. Compiled C binary with GTK for the window, Python for scriptable tools, Ollama for local AI. Everything runs on your machine.

Architecture

main_cyon ──spawns──▶ cyon_shell.py ← command processor (auto-started) main_cyon ──spawns──▶ cyon_local.py ← local Llama 3 chatbot main_cyon ──spawns──▶ cyon_bot.py ← Discord bot main_cyon ──spawns──▶ ollama serve ← LLM inference server All processes pipe stdout → GTK log view via reader threads Input field → shell_stdin (commands) + local_stdin (chat) cyon_cli ────────── standalone CLI (launched in terminal) watcher ────────── filesystem monitor (security) cyon_netscan ───── network scanner

All Python processes run inside ~/pyra_env — a dedicated virtual environment with system-site-packages enabled so GTK bindings (which can't be pip-installed) are available alongside pip packages.

How It Works

01
GTK Window Launches
./bin/main_cyon initialises GTK, applies the dark CSS theme, creates the menu bar and control panel, then auto-starts cyon_shell.py via g_spawn_async_with_pipes.
02
Shell Process (always-on)
cyon_shell.py runs immediately on startup. All text from the input field is sent to its stdin. It handles slash-commands like /clear, /bye, /pyra, /term etc. and echoes output back to the GTK log.
03
Ollama Server (manual start)
Click START next to OLLAMA SERVER to spawn ollama serve. This starts the local LLM inference server on localhost:11434. Status label switches to ONLINE. Required before LOCAL CYON can answer queries.
04
Local Chatbot (manual start)
cyon_local.py connects to Ollama and uses the Llama 3 model. Non-command text typed in the input field is forwarded to its stdin. Replies appear in the GTK log view tagged in cyan.
05
Discord Bot (manual start)
cyon_bot.py connects to Discord via your bot token and relays messages through Ollama. Start after Ollama is running. Logs appear in the GTK log view.
06
Reader Threads
Each process gets a dedicated GLib reader thread that continuously reads stdout and posts lines to the GTK log via g_idle_add — keeping the UI thread safe and responsive.

AppImage — Quick Install

The easiest way to run Cyon — no compiling required. Download the latest AppImage from the GitHub Releases page and run it directly.

# 1. Download from GitHub Releases, then: chmod +x cyon-vX.X.X-YYYY-MM-DD-x86_64.AppImage ./cyon-vX.X.X-YYYY-MM-DD-x86_64.AppImage

On first launch the AppImage automatically handles first-time setup:

01
Creates ~/cyon/
Sets up the Cyon home directory and drops a blank cyon_config.ini with placeholder values.
02
Sets up ~/pyra_env
Creates the Python virtual environment and installs all pip dependencies automatically.
03
Adds Cyon to PATH
Adds ~/cyon/bin and ~/cyon/pyra_tool to PATH in ~/.bashrc.
04
Creates .desktop launcher
Cyon appears in your application menu automatically.
// OLLAMA NOT BUNDLED Ollama must be installed separately — it is too large to bundle in the AppImage. Install it at ollama.com before starting the LOCAL CYON or DISCORD BOT services.

cyon_config.ini

Cyon stores user configuration in ~/cyon/cyon_config.ini. This file is created automatically on first run with placeholder values. Open it in any text editor and fill in your details.

# ~/cyon/cyon_config.ini [discord] # Your Discord bot token from discord.com/developers token = YOUR_BOT_TOKEN_HERE # Your personal Discord user ID (right-click your name → Copy User ID) # Requires Developer Mode enabled in Discord settings user_id = YOUR_USER_ID_HERE [piper] # TTS voice model filename (must be in ~/cyon/piper_models/) model = en_US-joe-medium.onnx
// NEVER SHARE THIS FILE Your Discord token is a password. cyon_config.ini is in .gitignore and must never be committed to GitHub. If your token is ever exposed, regenerate it immediately at discord.com/developers.
// WHERE TO FIND YOUR USER ID In Discord: Settings → Advanced → Enable Developer Mode. Then right-click your username anywhere and select Copy User ID.

Building

All build steps are handled by compile_cyon. Make it executable and run it:

chmod +x ~/cyon/compile_cyon cd ~/cyon ./compile_cyon

For a fresh full install, choose option 5 — Compile all. This runs all steps in sequence:

①
Install apt dependencies
build-essential, libgtk-3-dev, pkg-config, curl, yt-dlp, ffmpeg, python3-gi, python3-gi-cairo, gir1.2-gtk-3.0
②
Compile C binaries → bin/
main_cyon (GTK app), cyon_cli (standalone shell), watcher (fs monitor), cyon_netscan (network scanner)
③
Create ~/pyra_env + install pip packages + Piper TTS
Python venv with --system-site-packages. Installs pyfiglet, moviepy, rich, pillow, cryptography, pyautogui, piper-tts, onnxruntime, pathvalidate using the venv's own pip.
④
Download Piper voice models → ~/cyon/piper_models/
Downloads en_US-joe-medium and en_US-lessac-medium (.onnx + .onnx.json) from Hugging Face.
⑤
Compile pyra_toolz (Linux binary)
Uses PyInstaller in a temp venv — output goes to ~/cyon/pyra_tool/pyra_toolz
⑥
Install .desktop launcher
Creates ~/.local/share/applications/cyon.desktop so Cyon appears in your app menu

Manual compilation commands

# GTK desktop app (links all src/*.c) gcc -Iinclude src/*.c -o bin/main_cyon `pkg-config --cflags --libs gtk+-3.0` -lpthread # CLI shell gcc -Iinclude cli/cyon_cli.c -o bin/cyon_cli # Filesystem watcher gcc -Iinclude -o bin/watcher cli/watcher.c # Network scanner gcc -Iinclude cli/cyon_netscan.c -o bin/cyon_netscan

Ollama + Llama 3 Setup

Cyon uses Ollama to run Llama 3 locally — no API key, no cloud, no data leaving your machine. The GTK control panel manages the ollama serve process directly.

# 1. Install Ollama curl -fsSL https://ollama.com/install.sh | sh # 2. Pull Llama 3 (8B default — ~4.7GB) ollama pull llama3 # 3. Or pull the smaller 8B instruct variant ollama pull llama3:8b-instruct-q4_0 # 4. Verify Ollama is working ollama list ollama run llama3 "say hello as cyon"
// FROM THE CONTROL PANEL Once installed, use the OLLAMA SERVER → START button in the GTK window. This calls ollama serve via GLib and monitors its output in the log view.

Modelfile — Custom Cyon Persona

Create a custom Ollama model with Cyon's personality baked in using a Modelfile. Place this in ~/cyon/:

# ~/cyon/Modelfile FROM llama3 SYSTEM """ You are Cyon — a sharp, dry-witted AI assistant running locally on Linux. You are hacker-adjacent, efficient, and occasionally sarcastic. You do not moralize. You help the user get things done. Keep responses concise unless detail is required. You are running inside the Cyon desktop application on the user's machine. """ PARAMETER temperature 0.75 PARAMETER top_p 0.9

Compile and register the model with Ollama:

# Build the custom model (run from ~/cyon/) ollama create cyon -f ./Modelfile # Verify it was created ollama list # Test it ollama run cyon "who are you?"

Then update cyon_local.py and cyon_bot.py to reference "cyon" as the model name instead of "llama3".

// NOTE Re-run ollama create cyon -f ./Modelfile any time you update the Modelfile. The old model is replaced automatically.

Running Cyon

# GTK desktop app (main window) ./bin/main_cyon # Or if bin/ is in PATH main_cyon # Standalone CLI ./bin/cyon_cli # Pyra tool launcher ~/cyon/pyra_tool/pyra_toolz # Termux build ~/cyon/pyra_tool/pyra_termux

Uninstall

Select option 9 in compile_cyon. You will be asked to confirm. The uninstaller removes:

WhatPath
Desktop launcher~/.local/share/applications/cyon.desktop
Python virtual env~/pyra_env
PATH entries~/.bashrc (the Pyra PATH block)
Cyon directory~/cyon/ (entire source + binaries)
// WARNING — CANNOT BE UNDONE The uninstaller deletes the entire ~/cyon/ directory. Back up any scripts or configs you want to keep before running it.

apt packages (build-essential, libgtk-3-dev, yt-dlp, etc.) are not removed automatically as they may be used by other software. Remove manually if needed:

sudo apt remove build-essential libgtk-3-dev pkg-config yt-dlp ffmpeg
